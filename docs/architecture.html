Factory Inventory Management System

A full-stack demo application for inventory tracking, order management, demand forecasting, and spending analytics across factory operations. Vue 3 single-page app on the front, FastAPI service on the back, JSON files in memory as the data layer.

System Architecture

Three layers, one process per runtime, no database.

Data Flow

What happens when a user changes a filter on the Dashboard.

  1. User interaction
    The user picks a warehouse, category, status, or time period in FilterBar.vue. Selections write to refs in the useFilters composable, a singleton shared by every view.
  2. View reacts
    The active view (e.g. Dashboard.vue) watches the filters via getCurrentFilters() and triggers a reload. Loading and error states are tracked locally with refs.
  3. API call
    client/src/api.js serializes filters into URL query params, dropping any value of 'all', and issues a GET via axios to http://localhost:8001/api/<resource>.
  4. FastAPI receives
    Uvicorn dispatches to the matching route in main.py. Query params arrive as typed Optional[str] arguments; response_model documents the expected shape.
  5. Filter in memory
    Shared helpers apply_filters() and filter_by_month() walk the in-memory list, skipping any filter whose value is None or 'all'. Aggregation endpoints (dashboard summary, quarterly report) compute totals here too.
  6. Pydantic validates
    Each item is coerced against its model (InventoryItem, Order, DemandForecast, etc.) before serialization. Anything that doesn't fit becomes a 500 with a validation trace.
  7. Vue renders
    The response lands back in a ref. computed properties derive sorted/filtered slices and chart data; the template re-renders. Tables, KPIs, and custom SVG charts update in one tick.

Filters: The Unifying Pattern

Four filter dimensions are wired identically end-to-end; this is the project's most reused design pattern.

Time Period

Month (YYYY-MM) or quarter (Q1-2025). Server expands quarters to month lists via QUARTER_MAP.

Warehouse

Exact match against the warehouse field on each item or order.

Category

Case-insensitive match against category. Circuit Boards, Sensors, Actuators, Controllers.

Order Status

Case-insensitive match. Delivered, Shipped, Processing, Backordered.

Note: Inventory has no time dimension, so the month filter is ignored on /api/inventory. A value of 'all' (or absent) skips the filter on both sides.

Operational Notes

Running locally

Backend: cd server && uv run python main.py
Frontend: cd client && npm run dev
OpenAPI docs at localhost:8001/docs.

Persistence

There is none. All writes (purchase orders, tasks) live in process memory and disappear on restart. JSON files are read once at startup.

CORS & auth

allow_origins=["*"], no authentication. Local development only.

Reactivity contract

Views hold raw data in refs (e.g. allOrders); derived data — sorted lists, chart series, KPIs — live in computed properties so a filter change ripples through the UI in one tick.
